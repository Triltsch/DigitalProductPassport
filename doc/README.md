# Documentation

The `doc/` directory contains project documentation that supports planning, delivery, and future operations.

## Structure

- `planning/`: the primary project blueprint covering requirements, architecture, modules, roadmap, testing, operations, and risks.
- `seed/`: initial source material used to derive the planning package.

## Usage

- Start with `planning/README.md` for the navigation index.
- Treat the planning documents as the canonical source for repository structure and implementation sequencing.
